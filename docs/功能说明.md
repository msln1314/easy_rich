# Easy Rich 股票量化分析系统 - 功能说明

## 项目简介

Easy Rich 是一个基于前后端分离架构的股票量化分析平台，整合了股票数据获取、技术分析、预测建模、资讯阅读等功能。

- **前端技术栈**：Vue 3 + Element Plus + TypeScript + Vite
- **后端技术栈**：FastAPI + SQLAlchemy 2.0 + MySQL + APScheduler
- **数据来源**：AkShare（东方财富数据源）

### 1. 股票预测 (Prediction)

股票价格预测模块，基于机器学习模型对股票价格进行预测分析。

#### 功能特性
- **股票搜索**：支持按股票代码或名称模糊搜索
  - 单结果自动选择
  - 多结果弹出选择框
- **K线图展示**：展示股票历史价格走势（蜡烛图、成交量）
- **股票信息汇总**：展示股票基本信息、财务指标、技术指标
- **AI预测**：基于历史数据进行价格趋势预测
- **预测结果可视化**：展示预测曲线和置信区间

#### 使用方式
1. 在搜索框输入股票代码或名称（如：平安银行、000001）
2. 点击搜索或按回车键
3. 如果只有一个结果，自动加载该股票数据
4. 如果有多个结果，从弹窗中选择股票
5. 查看K线图和股票信息
6. 点击"开始预测"按钮生成预测结果

#### 关键文件
- 前端：`frontend/src/views/Vadmin/Stock/Prediction/Prediction.vue`

### 7. 定时任务 (Scheduled Tasks)

系统定时任务管理模块。

#### 功能特性
- **任务管理**：创建、编辑、删除定时任务
- **任务调度**：基于 Cron 表达式的任务调度
- **任务执行**：手动触发任务立即执行
- **执行日志**：查看任务执行历史和日志

#### Cron 表达式格式

系统支持 6 字段或 7 字段格式的 Cron 表达式：

**6 字段格式**：`秒 分 时 日 月 周`

```
0 0 2 * * ?      # 每天凌晨2点执行
0 */5 * * * ?    # 每5分钟执行一次
*/5 * * * * ?    # 每5秒执行一次
0 8 * * ?        # 每天8点执行
0 0 12 * * ?     # 每天12点执行
```

**7 字段格式**：`秒 分 时 日 月 周 年`

```
0 0 0 * * ? 2026    # 2026年每天午夜执行
0 0 2 * * ? *       # 每天凌晨2点执行（每一年）
```

#### 内置定时任务

**每日同步股票信息**
- 任务名称：每日同步股票信息
- 执行频率：每天凌晨2点执行
- 功能：从 AkShare 同步最新的股票基本信息、实时行情、行业板块数据
- Cron 表达式：`0 0 2 * * ?`

#### 使用方式
1. 进入系统管理 > 定时任务
2. 点击"新建任务"
3. 填写任务信息：
   - 任务名称
   - 触发器类型（cron/interval/date）
   - Cron 表达式或触发参数
   - 任务函数路径（如：`scheduler.main_day`）
4. 设置任务状态（启用/禁用）
5. 保存任务
6. 可手动执行或等待定时执行

#### 关键文件
- 后端：`backend/core/ap_scheduler.py`
- 后端：`backend/apps/admin/system/crud/task.py`
- 后端：`backend/utils/cron_util.py`

### 9. 条件选股策略 (Stock Selection Strategy)

基于自定义条件进行股票筛选和选股的策略模块。

#### 功能特性
- **多维度筛选**：支持根据多种技术指标和基本面指标进行筛选
  - 技术指标：MA、MACD、KDJ、RSI、成交量等
  - 基本面指标：市盈率、市净率、ROE、市值等
  - 价格指标：涨跌幅、价格区间、振幅等
- **组合条件**：支持多条件组合（AND/OR 逻辑）
- **策略保存**：保存常用的筛选条件为策略模板
- **策略管理**：创建、编辑、删除策略模板
- **快速选股**：一键执行策略，快速筛选符合条件的股票
- **结果导出**：将选股结果导出为 Excel

#### 策略示例

**均线金叉策略**
- 条件：5日均线上穿10日均线
- 用途：捕捉短期上涨趋势

**低估值策略**
- 条件：市盈率 < 15 且 市净率 < 1.5
- 用途：寻找被低估的优质股票

**主力资金流入策略**
- 条件：主力净流入 > 5000万 且 涨跌幅 > 0
- 用途：跟随主力资金动向

#### 使用方式
1. 进入条件选股策略页面
2. 点击"新建策略"，设置筛选条件
3. 配置筛选参数：
   - 选择指标类型（技术/基本面/价格）
   - 设置比较条件（大于/小于/等于/包含）
   - 输入目标数值或范围
4. 添加多个条件，选择逻辑关系（AND/OR）
5. 保存策略为模板
6. 点击"执行选股"进行筛选
7. 查看选股结果列表
8. 导出结果或将结果加入自选股

#### 关键文件
- 前端：`frontend/src/views/Vadmin/Stock/Analysis/components/AnalysisWrite.vue`
- 后端：`backend/apps/admin/stock/stock_analysis.py`
- 数据模型：`backend/apps/admin/stock/models/stock_analysis.py`

### 10. 监听策略 (Stock Monitoring Strategy)

实时监控股票价格变动，自动触发提醒或交易的策略模块。

#### 功能特性
- **价格监控**：设置股票价格的上限、下限，突破时自动提醒
- **指标监控**：监控技术指标（如 MACD 金叉、RSI 超买超卖等）
- **批量监控**：同时监控多只股票
- **策略管理**：创建、编辑、删除监控策略
- **启用/禁用**：灵活控制策略的启用状态
- **监控周期**：设置关注天数，自动过期历史监控
- **提醒方式**：支持系统通知、邮件等提醒方式

#### 监控条件类型

**价格条件**
- 价格上涨到：当价格达到或超过目标值时触发
- 价格下跌到：当价格达到或低于目标值时触发
- 价格区间：价格超出预设区间时触发

**涨跌幅条件**
- 单日涨幅超过：当日涨幅达到设定值时触发
- 单日跌幅超过：当日跌幅达到设定值时触发
- 连续N天上涨/下跌：连续涨跌达到N天时触发

**技术指标条件**
- 均线金叉/死叉：短期均线上穿/下穿长期均线
- MACD 金叉/死叉：DIF上穿/下穿DEA
- KDJ 超买/超卖：K、D、J值达到超买超卖区间
- RSI 超买/超卖：RSI值达到超买超卖区间

#### 使用方式
1. 进入监听策略页面
2. 点击"新建监控"，创建新的监控条件
3. 配置监控参数：
   - **监控名称**：为监控策略命名
   - **股票代码**：输入要监控的股票代码或名称
   - **监控条件**：选择触发条件类型和具体数值
   - **关注天数**：设置监控的有效天数
   - **备注**：添加备注说明策略用途
4. 保存监控策略
5. 启用/禁用监控（默认启用）
6. 系统实时监控股票行情
7. 当满足条件时，自动发送提醒通知
8. 查看监控历史记录

#### 监控状态
- **启用中**：策略正在执行监控
- **已禁用**：策略暂停监控
- **已触发**：条件已触发，监控完成
- **已过期**：监控超过关注天数，自动失效

#### API 说明

**创建监控策略**
```
POST /stock/stock_monitor_condition
参数：
- condition_content: 监控条件描述
- name: 监控名称
- tag: 股票名称
- user_id: 用户ID
- watch_days: 关注天数
- is_active: 是否启用
- remark: 备注
```

**获取启用的监控条件**
```
GET /stock/stock_monitor_condition/active?user_id={user_id}
```

**更新监控启用状态**
```
PUT /stock/stock_monitor_condition/{condition_id}/active
参数：
- is_active: 是否启用（1启用 0禁用）
```

## 快速开始

### 环境要求

- Python 3.10+
- Node.js 14.18.0+
- MySQL 5.7+
- Redis（可选）

### 后端启动

```bash
# 进入后端目录
cd backend

# 安装依赖
pip install -r requirements.txt

# 配置数据库连接（编辑 application/settings.py）

# 初始化数据库（首次使用）
python main.py init

# 启动服务
python main.py run
```

后端服务默认运行在：`http://localhost:9000`

### 前端启动

```bash
# 进入前端目录
cd frontend

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

前端服务默认运行在：`http://localhost:3000`

- `GET /stock/user_stock` - 获取用户自选股列表

- `GET /stock/stock_board_industry` - 获取行业板块数据
- `GET /stock/stock_board_concept` - 获取概念板块数据

- `POST /stock/stock_monitor_condition` - 创建监控策略

- `PUT /stock/stock_monitor_condition/{data_id}` - 更新监控策略

- `GET /stock/stock_monitor_condition/{data_id}` - 获取监控策略详情
- `GET /stock/stock_monitor_condition/active` - 获取启用的监控条件
- `PUT /stock/stock_monitor_condition/{condition_id}/active` - 更新监控启用状态
- `GET /stock/stock_monitor_condition/user/{user_id}` - 根据用户ID获取监控条件列表

## 开发指南

### 添加新功能模块

#### 后端添加 API

1. 在 `backend/apps/admin/` 下创建新模块目录
2. 创建必要的文件结构：
   ```
   新模块名/
   ├── __init__.py
   ├── models/       # 数据模型
   ├── schemas/      # Pydantic 数据模型
   ├── params/       # 查询参数
   ├── crud/         # 数据库操作
   └── views.py     # 视图函数
   ```

3. 在路由中注册新 API

#### 前端添加页面

1. 在 `frontend/src/views/Vadmin/` 下创建新模块目录
2. 创建 Vue 页面组件
3. 在菜单配置中添加页面路由
4. 如需 API，在 `frontend/src/api/` 下添加对应的 API 文件

```bash

```

## 常见问题

### Cron 表达式验证失败

确保 Cron 表达式符合以下格式：
- 6 字段：`秒 分 时 日 月 周`
- 7 字段：`秒 分 时 日 月 周 年`

示例：
- ✅ `0 0 2 * * ?` - 每天2点
- ❌ `0 2 * * *` - 缺少第6个字段
- ✅ `*/5 * * * * ?` - 每5分钟
- ✅ `0 */5 * * * ?` - 每5分钟（从0秒开始）

### 数据库连接失败

检查 `application/settings.py` 中的数据库配置：
- 数据库地址是否正确
- 用户名和密码是否正确
- 数据库是否已创建

### 前端无法连接后端

检查：
1. 后端服务是否正常运行
2. API 地址配置是否正确（`.env.development` 或 `.env.production`）
3. 跨域配置是否正确
